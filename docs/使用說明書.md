# NPDL 評量設計工作室｜完整使用說明書

適用對象：高中教師（含共同備課）  
正式測試站：[https://evan645test.github.io/npdl-assessment-studio/](https://evan645test.github.io/npdl-assessment-studio/)  
文件版本：對應單一流程合併版（課程＋評量連續步驟）  
一頁速查：[教師10分鐘速查卡.md](./教師10分鐘速查卡.md)

---

## 1. 這是什麼

本工具協助教師依 **108 課綱**、**NPDL 6Cs 子向度** 與 **學習設計四要素**，在**同一條流程**完成：

1. 先選 **已有／未有課程** → NPDL 構造（校準 → 終點 → 證據 → 課前 → 藍圖）  
2. 再於同頁完成 **課後遷移**（可填設計差異改寫）、匯出與 Google Forms  

核心理念是 **逆向設計**：先鎖定「學生最後要學會什麼、用什麼證據判斷」，再倒推節次與評量題目。

### 1.1 單一流程（不再分兩個工作區）

頂部不再切換「課程設計／評量設計」。上方分頁依序為：校準 → 學習終點 → 評量證據 → 診斷題組 → 節次與交付 → **課後與匯出**。

舊網址 `/course-ideation/` 與 `?workspace=assessment` 都會回到合併首頁。

**已移除**：不經課程構造、直接從零產生診斷＋遷移整包的獨立評量路徑。

### 1.2 建議主路徑（第一次使用請照此走）

```text
設定 AI 金鑰
  → 選擇「已有課程」或「未有課程」
  → 已有：上傳／貼上教案 → 轉換並帶入 → 補齊關鍵字分析
     未有：填課／分析關鍵字
  → 校準與選擇（108 課綱＋6Cs）
  → 學習終點（生成並確認）
  → 評量證據（建立並確認）
  → 課前題組（含課後 plannedPost 對齊）
  → 節次與交付（填限制 → 產生藍圖 → 確認）
  → 進入課後與匯出
  → 產生課後評量（可填設計差異後改寫）
  → 匯出或建立 Google Forms
```

## 2. 開始前準備

### 2.1 瀏覽器與裝置

- 建議使用最新版 Chrome、Edge 或 Safari  
- 桌面與手機皆可操作；複雜編修建議桌面  
- 資料存在**目前這台裝置的瀏覽器**（localStorage），換裝置或清除網站資料會遺失

### 2.2 AI 設定（必做）

1. 點右上角 **齒輪**，開啟「NPDL Studio AI 設定」  
2. 選擇供應商與模型：  
   - **Gemini（Google）**：預設 **Gemini 2.5 Flash**  
   - **ChatGPT（OpenAI）**  
   - **Grok（xAI）**  
3. 貼上對應 **API Key**  
4. 可選「測試 AI 連線」確認可用  
5. 按「完成設定」

| 項目 | 說明 |
|------|------|
| 金鑰存放 | 僅存於目前瀏覽器，**未加密明文** |
| 清除 | 「清除此金鑰」或「清除所有 API Key」 |
| 合併版 | **不提供** Puter 免費模型；需自備金鑰（BYOK） |

**安全提醒**：公用電腦用完請清除金鑰。金鑰不會寫進提示詞、草稿或交接檔。

### 2.3 首次呼叫 AI 的同意畫面

第一次在課程端呼叫 AI 時，系統會顯示：

- 供應商、模型、用途  
- **即將傳送的完整內容**

請閱讀後按「同意並繼續」。同意會記在本瀏覽器，之後不逐次詢問；可隨時撤回。

### 2.4 隱私與資料原則（請務必遵守）

- **不儲存學生姓名**；正式流程只用班級代號、座號等非姓名識別  
- 上傳教案前，請自行移除學生個資、家長聯絡方式等敏感內容  
- 原始教案檔、檔名與完整擷取文字**不會**被系統保存  
- NotebookLM **不會**在瀏覽器執行時連線（僅開發期使用）

## 3. 介面導覽（如何看、如何點）

### 3.1 頂部「目前流程下一步」（主 CTA）

畫面上方的 **NextActionPanel** 是全頁主按鈕：告訴你「現在該做哪一步」。  
多數「純產生」動作（例如第一次校準、生成學習終點）請優先按這裡。

### 3.2 設計分頁（我在哪）

課程設計內有五個分頁，代表階段位置，**不是**第二套重複主流程：

1. 校準與選擇  
2. 學習終點  
3. 評量證據  
4. 課前題組  
5. 節次與交付  

### 3.3 按鈕前後位置規則（固定）

| 類型 | 條件 | 按鈕位置 |
|------|------|----------|
| **純產生** | 不必先在本區填新欄位，按下即產生 | **在前**：說明 → 按鈕／上方提示 → 產出區 |
| **先填後產** | 必須先填寫或勾選再產生 | **在後**：欄位 → 按鈕 → 結果 |
| **審閱確認** | 先看 AI 結果再鎖定 | **在後**：內容 → 確認鈕（次要邊框樣式） |

分頁內**不會**再放一顆與頂部主 CTA 同文案的主色大鈕搶視線；若屬純產生，區內多半是「請先用上方主按鈕完成……」的短提示。  
例外：節次藍圖屬「先填後產」，填完限制欄位後，欄位下方會有「AI 產生單元節次藍圖」。

## 4. 課程設計｜逐步操作

左側／側欄可展開課程輸入；上方依流程往下走。下列依建議順序說明。

### 4.1 選擇方向、課程輸入與關鍵字分析

**你要做的事**

1. 先選 **已有課程**（轉換為 NPDL 敘述）或 **未有課程**（關鍵字發想）  
2. 填寫：年級、學科、單元名稱、教學主題  
3. 填寫 **3–5 個核心關鍵字**  
4. （選用）從測試範例載入：地理、化學、生物、物理、數學、國文、英文、社會  
5. **已有課程路徑**：上傳教案附件或貼上文字 → AI 轉換 → **你勾選** →「帶入並繼續 NPDL 構造」

**教案轉換規則（已有課程）**

| 項目 | 限制 |
|------|------|
| 格式 | 文字型 PDF、DOCX、UTF-8 TXT，或直接貼上文字 |
| 檔案大小 | 上限 10 MB |
| 送 AI 文字 | 上限約 40,000 字元 |
| 帶入方式 | AI 先整理結構化摘要 → **你勾選** →「帶入並繼續 NPDL 構造」才進入後續 |

未有課程路徑不會顯示教案轉換區塊；若要改方向，按「更改方向」（會清空下游分析）。

**主按鈕**

- 頂部主 CTA：**開始分析課程關鍵字**  
- 側欄亦可：**AI 分析核心關鍵字**（屬先填後產，欄位在前、按鈕在後）

**產出**：主題群、摘要與後續素養對齊訊號（階段一）。

**可行性預設（系統假設）**

- 台灣高中約 30–40 人班級  
- 約 50 分鐘課時  
- 一般教室、少量共用裝置  
- 超出現場條件的構想會縮小為校內、公開資料、模擬或低科技版本  
- 理化生地科可能附「一般高中可能具有的基礎器材」清單，**一律需教師確認數量、狀態與安全**；並附無器材時的替代方案

---

### 4.2 校準與選擇（108 課綱＋6Cs）

**你要做的事**

1. 用上方主按鈕完成初次校準（區內會提示「請先用上方主按鈕完成校準」）  
2. 檢視學習表現、學習內容推薦（系統各推約 5 項，並採用最相關者）  
3. 教師從推薦或分類清單中，**各選 2 項**學習表現／學習內容  
4. 選定 **1–2 項** 6Cs 子向度（顯示正式四級學習進程）  
5. 若改選後需重校準：選取區在前 → 按 **「依教師調整重新校準」**（在兩欄選取之後）

**主按鈕文案**

| 時機 | 文案 |
|------|------|
| 初次 | **進行 108 課綱與 6Cs 校準**（頂部主 CTA） |
| 改選後 | **依教師調整重新校準**（選取區下方） |

**產出**：課綱對齊、6Cs 推薦、三層學習成果草稿（知識基礎／素養子向度／四要素整合實踐）。

**教師自訂**：若受控課綱清單沒有合適條目，可加入明確標示為「教師自訂／未由系統核對」的依據。

**單卡修改**：內容卡可使用「AI 修改這張卡」→ 預覽 patch → 套用；只改該卡，不會連帶改寫同階段其他卡片。

> 改選但尚未重新校準時，下游階段會保留為舊版本，**重新校準後才更新**。

---

### 4.3 學習終點

對應逆向設計的 Desired Results。

**尚未生成（純產生）**

- 依上方主 CTA **生成學習終點**  
- 區內若保留動作，會在說明之後、結果之前

**已有結果（審閱確認）**

審閱下列內容後，在區塊底部按確認：

- 遷移目標  
- 核心理解、核心問題  
- 成功指標（具穩定 ID）  
- 三層學習成果與 NPDL 四要素說明  

**確認鈕**：**確認學習終點，開放評量證據**

> 教師確認前，不會開放下游評量證據。

**NPDL 學習設計四要素（固定）**

1. 學習夥伴關係  
2. 學習環境  
3. 數位利用  
4. 教學實踐  

（不使用含義不同的「4E 教學循環」用語。）

---

### 4.4 評量證據

**尚未建立（純產生）**

- 依上方主 CTA：**AI 建立完整評量證據**  
- 區內先看「請用上方主按鈕建立」類提示，**不要**期待先出現大片空編輯區

**已有證據（審閱確認）**

審閱並可逐項編修：

- 真實總結任務（預設 3–5 人小組、正常課時、校內受眾、低成本／一般設備；特殊條件須附低科技替代）  
- 課前／課後 Q1–Q4 的**評量目的與預期證據**（此處是證據設計，不是正式題幹定稿處）  
- 逐節形成性證據（含後續教學決策）  
- 學科四級規準  

**確認鈕**：**確認證據，前往課程敘述與課前評量**

**重新建立**：次要樣式，放在標題列附近或確認鈕旁，不與首次產生搶主位。

> 課程工具**不撰寫正式評量題目**。正式 Q1–Q4 在「課前題組」種子與評量工作區完成。  
> 解鎖重編證據會讓下游藍圖與 Canvas 提示詞過期，需重新確認／產生。

---

### 4.5 課前題組

**尚未建立**：提示在前 → 用上方主按鈕產生。

**主按鈕**：**AI 產生課程敘述與完整課前評量**（可重新產生並校準）

**產出**

- 四級課程敘述語  
- 正式課前 Q1–Q4  

進入「課後與匯出」後，會**唯讀沿用**這份課前種子；課後遷移必須對齊 plannedPost。

**Q1–Q4 能力階梯（固定）**

1. 概念理解  
2. 行動應用  
3. 生活遷移  

---

### 4.6 節次與交付（先填後產）

#### A. 產生單元節次藍圖

1. 填寫限制欄位：總節數、每節分鐘、必須保留的活動、設備場地、先備經驗、差異化需求等  
2. 在**欄位下方**按 **「AI 產生單元節次藍圖」**（頂部主 CTA 也可觸發同一動作）  
3. 審閱逐節藍圖（每節連結成果、成功指標、證據、6Cs、四要素，並含學習單設計 brief）  
4. 按 **「確認藍圖並更新／啟用 Canvas」**

#### B. Gemini Canvas 完整提示包

確認藍圖後，交付中心可：

| 動作 | 說明 |
|------|------|
| 預覽完整提示詞 | 檢視即將貼到 Gemini 的內容 |
| 複製完整單元 | 複製到剪貼簿 |
| 下載 Markdown | 下載提示包 |
| 開啟 Gemini | 開啟 `https://gemini.google.com/app`（**不帶**任何專案資料或 query） |
| 標記外部產生 | 標記你已在外部完成產生 |

**重要限制**

- App **不**在站內編輯完整逐節教案或學習單  
- App **不**提供跨站自動貼上；請自行貼到 Gemini Canvas  
- 教師用自己的 Google 帳號操作 Gemini，並在 Gemini 內修改產出  
- 上游課綱／終點／證據／限制變更後，藍圖會標示需重新校準，正式提示詞會停用，直到重新產生並確認

#### C. 進入課後與匯出

- **下載課前 Markdown**、**列印／另存 PDF**  
- **進入課後評量與匯出** → 確認對話 → **確認，進入課後與匯出**（同頁切到「課後與匯出」分頁）

會帶入／沿用：年級、學科、課程名稱、課綱原文、學習成果、成功指標、證據工具、選定子向度、唯讀敘述語與課前 Q1–Q4、預定課後證據目的、節次藍圖等。

**已移除**：不經上述步驟、直接從零產生整包診斷＋遷移的獨立評量路徑。

## 5. 課後與匯出｜逐步操作

此步驟在課程流程**同一頁**，不另開工作區。需先有課前種子；否則請回到「診斷題組」產生。

### 5.1 產生／改寫課後遷移

1. 確認敘述語與課前 Q1–Q4 已唯讀顯示（課後會依 plannedPost 對齊）  
2. （選填）填寫「設計差異說明（用於改寫課後）」——**不得**用來改寫 plannedPost 的 purpose／criterionIds  
3. 未填差異：按 **「產生課後評量」**；有填差異：按 **「依設計差異改寫課後評量」**  
4. 檢視課後遷移題組與整體文件（三模組預覽）  
5. 需要時可 AI 微調後「套用微調」、收藏題庫、切換裝置寬度預覽  

設計差異**不會**自動覆寫已確認的終點、證據或節次藍圖。

### 5.2 Google Forms

每份表單結構：

- 完整共用情境  
- Q1–Q3：三題必填單選  
- Q4：概念理解／行動應用／生活遷移，三題必填長答  
- 建立後提供教師編輯連結與學生作答連結  

操作：

1. 開啟 Google Forms 相關設定（OAuth Client ID；正式站通常已由部署者設定）  
2. 登入 Google 並建立課前／課後問卷  
3. 若中途失敗：用 **「重試未完成問卷」**——只接續未完成階段，**不會**重建已完成表單或重複寫題  

建立流程：建立空表單 → 寫入題目 → 發布並開啟回應。

> 目前**尚未**支援整批讀取 Forms 回應與 AI 自動判讀；欄位命名已預留供後續版本。

### 5.3 匯出

| 功能 | 說明 |
|------|------|
| 完整課程與評量證據包 | 下載 Markdown、列印／另存 PDF |
| 複製題目 Markdown | 單份題組文字 |
| 收藏至題庫 | 存於本機瀏覽器題庫 |

## 6. 常見問題（FAQ）

### Q1. 為什麼畫面上有兩個地方好像都能「產生」？

上方是**全頁下一步**；分頁內只在「先填後產」或「確認鎖定」時放對應按鈕。若區內寫「請用上方主按鈕……」，請按上方，不要找第二顆同文案主色大鈕。

### Q2. 改了課綱選項，後面怎麼沒變？

改選後必須按 **「依教師調整重新校準」**。未重校準前，下游會刻意保留舊版本。

### Q3. 為什麼藍圖／Canvas 不能用了？

上游（課綱、學習終點、證據、課程限制）變更後，下游會過期。請依提示重新產生並再次確認。

### Q4. 金鑰安全嗎？

金鑰只存在本機瀏覽器且未加密。請勿在公用電腦長期保留；用畢請清除。系統不會把金鑰寫進匯出檔或交接資料。

### Q5. 可以存學生姓名嗎？

不可以。請只用班級代號與座號等非姓名識別。

### Q6. 換電腦資料還在嗎？

預設不會。資料在瀏覽器本機。請自行下載 Markdown／PDF，或進入課後步驟前先匯出備份。

### Q7. Gemini 產生的教案可以在 App 裡改嗎？

不行。請在 Gemini Canvas 中修改。App 負責組裝提示詞與課程／評量結構化產物。

### Q8. 108 課綱資料從哪來？

建置期自國教院正式領域／科目資料整理為受控快照。同碼原文衝突的條目會排除。無法對應時可標註「教師自訂／未由系統核對」。

## 7. 故障排除

| 狀況 | 建議處理 |
|------|----------|
| AI 呼叫失敗 | 檢查供應商、模型、金鑰是否正確；用「測試 AI 連線」；確認額度與網路 |
| 看不到評量證據步驟 | 是否已「確認學習終點」 |
| 看不到課前題組 | 是否已「確認證據」 |
| Canvas 按鈕停用 | 是否已確認藍圖；上游是否已過期需重產 |
| Google Forms 失敗 | 確認已登入授權；用「重試未完成問卷」；部署站是否已設定 OAuth Client ID |
| 畫面像舊版 | 強制重新整理（清除快取）；確認網址為合併版根路徑 |
| 手機難操作 | 改用桌面完成複雜選取與長文審閱 |
